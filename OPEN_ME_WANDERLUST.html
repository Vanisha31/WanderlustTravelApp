This file explains what was added and how to see the live app locally before pushing to GitHub or deploying.

Run Locally

cd "C:\Users\Vanisha\OneDrive\Attachments\Desktop\Documents\New project\WanderlustTravelApp"
npm install
npm run dev

Then open:

  • Frontend app: http://localhost:5173/
  • Backend health check: http://localhost:4000/api/health
  • Saved bookings API: http://localhost:4000/api/bookings

What Is Now Real

  • React + Vite frontend like a travel booking portal.
  • Node.js REST backend in server/index.js.
  • Bookings API: GET /api/bookings and POST /api/bookings.
  • Saved bookings file: server/data/db.json.
  • Checkout form saves traveler details, selected seats, route, amount, and booking status.
  • My Trips dashboard shows bookings returned by the backend.

Deploy Note

Vercel is perfect for the frontend. For public real-user bookings, deploy the backend on Render/Railway or connect a Vercel serverless API to a hosted database like Neon/Supabase PostgreSQL.

Open LIVE_DEPLOYMENT_GUIDE.md in this folder for the exact GitHub, Vercel, and backend deployment steps.
